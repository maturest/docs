一套接口应尽量满足：1、安全，高效，易扩展。2、简单明了，可读性强，没有歧义。3、API 风格统一，调用规则，传入参数和返回数据有统一的标准。

Restful API 设计规范：

  • Https 为接口的安全提供保障。
  • 域名，应当尽可能的将 API 与其主域名分开。
  • 版本控制，版本迭代，有的提供不兼容的版本，为了使旧用户可以正常使用，需要控制好 API 版本。
  • 用 URL 定位资源，大部分命名格式为名词的复数形式。尽量不要在 URL 中出现动词。
  • 用 HTTP 动词描述操作，比如 GET、POST、PUT、PATCH、DELETE。
  • 资源过滤，提供查询参数，比如分页，排序，过滤等。
  • 正确使用状态码，可以让响应数据更具可读性。
  • 数据响应格式，JSON 是最常用的，但 XML 也有其优势，比如 XML 格式的数据更容易被第三方工具解析。
  • 调用频率限制，限制调用频率，避免被攻击。
  • 编写文档，提供 API 文档，让用户了解 API 的用法，并方便进行二次开发。
