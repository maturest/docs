# OSI七层参考模型

高层：负责主机之间的数据传输（7,6,5） 底层：负责网络数据传输（3,2,1）

7、应用层（提供应用程序间通信） 6、表示层（处理数据格式、数据加密） 5、会话层（建立、维护和管理会话） 4、传输层（建立主机端到端连接，电脑里面的应用） 3、网络层（寻址和路由选择） 2、数据链路层（提供介质访问，链路管理等，网卡） 1、物理层（比特流传输，网线）

# TCP/IP 协议栈

应用层（提供应用程序网络接口） 传输层（建立端到端的连接） 网络层（寻址和路由选择） 数据链路层（物理介质访问） 物理层（二进制数据流传输）

各层实现的协议：

应用层： HTTP:超文本传输协议，基于TCP,使用端口80 SMTP:简单邮件传输协议，基于TCP,使用25端口 FTP:文件传输协议，基于TCP,数据端口20，控制端口21 TELNET:远程登陆协议，基于TCP,使用23端口 DNS:域名解析，基于UDP,使用53号端口 SSH:安全外壳协议，基于TCP,使用22端口

传输层： TCP:传输控制协议，一种面向连接的，可靠的，基于字节流的传输层通信协议。 UDP:用户数据报协议，一种面向无连接的通讯协议，不可靠的，基于报文的传输层通信协议。 SCTP:流量传输控制协议，一种面向连接的传输协议 MPTCP:多路径传输控制协议

网络层： IP:internet协议。 ICMP:internet控制报文协议。用于在主机、路由器之间传递控制消息，检测网络痛不痛，主机是否可达、路由是否可用等网络本身的消息。 ARP:地址解析协议，通过目标设备的IP地址，查询目标设备的MAC地址，以保证通信的顺序进行。 RARP:反向地址解析协议。

# HTTP请求格式

  • 请求行：请求方法，请求URI,http-version,CRLF(换行)

  • 消息报头: 包含一系列的键值对

  • 请求正文（正文）：注意喝消息报头之间有一个空行

# 请求方法

  • get

  • post

  • put

  • delete

  • head

  • options

# HTTP响应格式

  • 状态行：http版本，状态码，状态描述，以空格分隔

  • 响应头：消息报头，包含一系列的键值对

  • 响应正文：返回内容，注意和响应头之间有一个空行

# HTTP状态码

1xx 请求已被服务接手，继续处理

2xx 请求已成功被服务器接收，理解、并接受

3xx 重定向

4xx 请求错误

5xx 服务器错误

# 宏观解析HTTP数据流的传输过程

1、打开浏览器，输入网址

2、浏览器先尝试从 host 文件中获取的刚输入的网址对应的IP地址，如果能取到则直接使用 hosts 文件的解析结果。

3、如果不能，就是用DNS协议来获取IP地址，在DNS协议中，PC会想你的本地DNS服务器求助，请求DNS服务器之后，得到IP地址。

ipconfig /all

4、浏览器会请求获得的IP地址对应的服务器，web服务器收到请求并做响应处理，将请求的内容返回给客户端浏览器。

5、浏览器收到回复，是以html形式的文本，并渲染出来。

# 微观解析 HTTP数据流的传输过程

  • 域名解析

  • 建立连接（TCP三次握手）

客户端发送数据包，问服务器我可以和你建立连接吗？ 服务端收到数据包，回答可以啊 客户端就与服务器建立连接

  • 发送http请求

  • 断开连接（TCP的四次挥手）

客户端发送数据包告诉服务器：我没什么要请求的了，但我还可以接收你的回复，如果没什么回复的话，可以断开连接吗？

服务端收到数据包说：我没什么回复了，可以关闭连接了，但我还没准备好，等一下我告诉你吧

服务端向客服端发送数据包：我准备好关闭连接了。

客户端发送关闭连接的数据包。
